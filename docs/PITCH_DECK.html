The Gap

"Ready for production?" is still answered by gut feel.

The Problem
  • Readiness reviews are run by hand by principal and staff engineers — slow, subjective, inconsistent.
  • AI agents and MCP products now ship faster than teams can assess them.
  • There's no shared standard for what "production-ready" even means.
  • Ask ChatGPT and it flatters you — it won't tell you no.
The Solution
  • A deterministic engine extracts evidence from your doc and scores 8 axes — same input, same verdict, every time.
  • Every score cites the exact evidence behind it: what's present, what's missing.
  • Synthesized from 12 industry frameworks (Google SRE, Amazon ORR, DORA, NASA TRL, OWASP AI…).
  • It is willing to say NO — an honest verdict, not a pep talk.

How it works

Read document → Extract evidence signals → Score 8 axes → Verdict + TRL + roadmap
🧮

Deterministic, not a wrapper

Pure-logic core with negation awareness — "no distributed tracing" correctly counts as a gap. Zero dependencies, zero data leaving the browser.

🔍

Evidence chips + gap report

Found/missing chips cite real evidence, benchmarked against industry standards, plus a remediation roadmap and auto-written exec narrative.

🧭

Adaptive framework

Auto-selects by product type: AI Agent, SaaS/API, Consumer/Marketplace, or General. The 8-axis model is the NPS of production readiness.

🤖

MCP server

Exposed as an assess_readiness tool — Claude and other agents can call PilotIQ directly as part of their workflow.

🔌

Provider-agnostic LLM

Optional LLM layer (Mock / OpenAI / Anthropic) via strategy pattern. The deterministic core runs perfectly with it switched off.

📤

Radar + exports

Per-axis radar chart, NASA TRL level 1–9, and one-click JSON / Markdown export for review docs and audits.

Built to Ship

System design, market & scale

System Design
  • Clean layered / hexagonal design: pure core/ (no React), optional llm/, features/ UI.
  • Config-driven frameworks — add a framework without touching engine code.
  • 13 passing Vitest tests. React 18 · Vite 6 · Tailwind 4 · Recharts.
Market & GTM
  • Buyer: VP Engineering / platform teams. User: principal & staff engineers.
  • Why now (2026): agents ship faster than teams can vet — no shared standard exists.
  • PLG: free assessments → team workspaces → benchmark-data network effect.
Scale & Production
  • Stateless deterministic engine — scales infinitely as a static app.
  • ~$0.02–0.05 per assessment in optional LLM mode; repository pattern swaps in-memory for Redis/Postgres.
  • PilotIQ assessed itself → CONDITIONAL, TRL 6. We dog-food it.
